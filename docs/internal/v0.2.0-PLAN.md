# Version 0.2.0 - Touch Input & Interactivity

**Timeline:** 2-3 weeks
**Status:** Planning
**Theme:** Making the device truly interactive

---

## Overview

Version 0.2.0 focuses on making the Nexus device interactive by integrating touch input and adding multiple display modes. Currently, the device is a passive display - it shows information but users can't interact with it. This release changes that.

---

## Goals

1. ✅ Users can interact with the device via touch gestures
2. ✅ Device responds to touch with configurable actions
3. ✅ Multiple display modes users can cycle through
4. ✅ Complete system monitoring (add CPU load)
5. ✅ Touch configuration via Flutter UI

## Risk Assessment

### High Risk
- **Touch hardware compatibility** - May behave differently across devices
  - *Mitigation:* Test on multiple devices if possible, add debug logging

- **Performance impact** - Polling touch + mode switching may impact rendering
  - *Mitigation:* Profile performance, optimize if needed

### Medium Risk
- **Configuration complexity** - Touch config may confuse users
  - *Mitigation:* Good defaults, clear UI labels

- **Mode rendering bugs** - Different modes may have rendering issues
  - *Mitigation:* Unit tests for each mode, manual testing

### Low Risk
- **API breaking changes** - New endpoints won't break existing clients
- **Backwards compatibility** - Config file is additive
