# Release Checklist for Nexus Open

This document outlines the steps to create and publish a new release.

## Pre-Release Testing

### Local Testing

- [ ] **Run full test suite**
  ```bash
  go test -v -race -coverprofile=coverage.out ./...
  go tool cover -func=coverage.out | tail -1
  # Ensure coverage is ≥60% (target: 64.9%)
  ```

- [ ] **Build binary locally**
  ```bash
  CGO_ENABLED=1 go build -v -o nexus-open \
    -ldflags "-X main.version=1.0.0 -X main.commit=$(git rev-parse --short HEAD)" \
    ./cmd/nexus-open
  ./nexus-open --version
  ./nexus-open --help
  ```

- [ ] **Test with mock device**
  ```bash
  # Run briefly to verify startup
  timeout 5s ./nexus-open || true
  # Check that config was created
  ls -la ~/.config/nexus-open/config.yaml
  ```

- [ ] **Verify API is working**
  ```bash
  # In one terminal
  ./nexus-open &
  sleep 2
  # In another terminal
  curl http://localhost:1985/api/health
  curl http://localhost:1985/api/config
  # Cleanup
  pkill nexus-open
  ```

### Package Testing

- [ ] **Build DEB package**
  ```bash
  ./scripts/build-deb.sh
  # If dpkg-deb is available:
  ls -lh dist/nexus-open_1.0.0_amd64.deb
  # Otherwise verify structure:
  ls -R build/deb/nexus-open_1.0.0_amd64/
  ```

- [ ] **Verify DEB contents**
  ```bash
  # Check files are in place
  test -f build/deb/nexus-open_1.0.0_amd64/usr/bin/nexus-open
  test -f build/deb/nexus-open_1.0.0_amd64/DEBIAN/control
  test -f build/deb/nexus-open_1.0.0_amd64/DEBIAN/postinst
  test -x build/deb/nexus-open_1.0.0_amd64/usr/bin/nexus-open
  ```

- [ ] **Validate packaging files**
  ```bash
  # Desktop file
  desktop-file-validate packaging/desktop/nexus-open.desktop

  # Flatpak manifest
  python3 -c "import yaml; yaml.safe_load(open('packaging/flatpak/com.github.nexusopen.NexusOpen.yaml'))"

  # Snap manifest
  python3 -c "import yaml; yaml.safe_load(open('packaging/snap/snapcraft.yaml'))"
  ```

### Hardware Testing (If Device Available)

- [ ] **Test with real Corsair iCUE Nexus**
  ```bash
  # Check device is visible
  lsusb | grep 1b1c:1b8e

  # Run application
  ./nexus-open

  # Verify:
  # - Display shows time
  # - Temperature readings appear
  # - Network stats update
  # - Weather fetches (after 30s)
  ```

- [ ] **Test configuration changes**
  ```bash
  # Edit config
  nano ~/.config/nexus-open/config.yaml
  # Change location, time format, colors
  # Verify changes take effect without restart
  ```

- [ ] **Test API configuration**
  ```bash
  # Update via API
  curl -X POST http://localhost:1985/api/config \
    -H "Content-Type: application/json" \
    -d '{"location":"San Francisco, CA","time_format":"24h","unit":"metric"}'

  # Verify changes on device
  ```

## Version Management

- [ ] **Update version numbers**
  - [ ] Update VERSION in build scripts if needed
  - [ ] Verify version is passed via -ldflags in build commands
  - [ ] Update PROJECT_PLAN.md status if needed

- [ ] **Create/Update CHANGELOG.md**
  ```bash

  # Follow Keep a Changelog format
  ```

- [ ] **Commit all changes**
  ```bash
  git add .
  git commit -m "Prepare v1.0.0 release"
  git push origin main
  ```

## Creating the Release

### Git Tag

- [ ] **Create and push tag**
  ```bash
  git tag -a v1.0.0 -m "Release v1.0.0"
  git push origin v1.0.0
  ```

### Build Release Artifacts

- [ ] **Build all packages**
  ```bash
  # Clean previous builds
  rm -rf build/ dist/

  # Build DEB
  ./scripts/build-deb.sh

  # Build AppImage (if tools available)
  ./scripts/build-appimage.sh
  ```

- [ ] **Create checksums**
  ```bash
  cd dist/
  sha256sum nexus-open_1.0.0_amd64.deb > SHA256SUMS
  sha256sum nexus-open-1.0.0-x86_64.AppImage >> SHA256SUMS
  cat SHA256SUMS
  ```

### GitHub Release

- [ ] **Create GitHub release**
  - Go to: https://github.com/mantonx/nexus-next/releases/new
  - Tag: v1.0.0
  - Title: "Nexus Open v1.0.0"
  - Description: Copy from CHANGELOG.md

- [ ] **Upload artifacts**
  - [ ] nexus-open_1.0.0_amd64.deb
  - [ ] nexus-open-1.0.0-x86_64.AppImage
  - [ ] SHA256SUMS
  - [ ] Source code (auto-generated by GitHub)

- [ ] **Mark as latest release**

## Distribution Packaging

### AUR (Arch Linux)

- [ ] **Update PKGBUILD**
  ```bash
  cd packaging/arch/
  # Update pkgver, sha256sums
  # Test build locally if possible:
  makepkg -si
  ```

- [ ] **Generate .SRCINFO**
  ```bash
  makepkg --printsrcinfo > .SRCINFO
  ```

- [ ] **Submit to AUR**
  ```bash
  # Clone your AUR repository
  git clone ssh://aur@aur.archlinux.org/nexus-open.git aur-nexus-open
  cd aur-nexus-open

  # Copy files
  cp ../packaging/arch/PKGBUILD .
  cp ../packaging/arch/.SRCINFO .

  # Commit and push
  git add PKGBUILD .SRCINFO
  git commit -m "Update to v1.0.0"
  git push
  ```

### Flathub

- [ ] **Submit to Flathub**
  1. Fork https://github.com/flathub/flathub
  2. Create PR with:
     - packaging/flatpak/com.github.nexusopen.NexusOpen.yaml
     - packaging/flatpak/com.github.nexusopen.NexusOpen.metainfo.xml
  3. Wait for review
  4. Address any feedback
  5. Wait for merge and publication

### Snap Store

- [ ] **Build Snap**
  ```bash
  cd /path/to/nexus-open
  snapcraft --use-lxd
  # OR
  snapcraft remote-build
  ```

- [ ] **Test Snap locally**
  ```bash
  sudo snap install --dangerous nexus-open_1.0.0_amd64.snap
  sudo snap connect nexus-open:raw-usb
  snap services nexus-open
  ```

- [ ] **Upload to Snap Store**
  ```bash
  snapcraft login
  snapcraft upload nexus-open_1.0.0_amd64.snap --release=edge
  # Test edge channel
  snapcraft release nexus-open 1 stable
  ```

  - [ ] Hacker News (if significant)
  - [ ] Twitter/Mastodon

### Monitoring

- [ ] **Monitor for issues**
  - [ ] Watch GitHub issues
  - [ ] Monitor AUR comments
  - [ ] Check Flathub/Snap Store reviews

- [ ] **Respond to feedback**
  - [ ] Fix critical bugs quickly
  - [ ] Consider patch release if needed

## Rollback Procedure

If critical issues are discovered:

1. **Unpublish from stores**
   ```bash
   # Snap: Revert to previous version
   snapcraft release nexus-open <previous-revision> stable

   # Flathub: Submit PR to revert commit
   ```

2. **Mark GitHub release as pre-release**
   - Edit release on GitHub
   - Check "This is a pre-release"

3. **Communicate the issue**
   - Post issue on GitHub
   - Update release notes

4. **Fix and create patch release**
   - Fix the issue
   - Create v1.0.1 with fix
   - Follow this checklist again
